# Райтап: XOR Decryption — SplitCTF

---

## Дано

В задании предоставлен файл с бинарными данными.  
Из описания и поведения файла можно сделать вывод, что данные были зашифрованы с помощью операции **XOR**.

Формат флага стандартный для соревнования:

```
SplitCTF{...}
```

---

## Анализ

XOR — симметричная операция, то есть:

```
A XOR B XOR B = A
```

Это означает, что при повторном применении того же ключа можно восстановить исходные данные.

В CTF-задачах чаще всего встречаются два варианта:
- XOR с одним байтом (single-byte XOR)
- XOR с повторяющимся ключом (repeating-key XOR)

Так как формат флага известен, можно использовать **known-plaintext attack**.

---

## Идея решения

1. Предположить, что внутри расшифрованного файла есть строка `SplitCTF{`.
2. Использовать её как известный фрагмент открытого текста.
3. Восстановить XOR-ключ.
4. Применить ключ ко всему файлу и получить читаемый текст.
5. Извлечь флаг.

---

## Пример решения (Python)

```python
data = open("cipher.bin", "rb").read()
known = b"SplitCTF{"

key = bytes([data[i] ^ known[i] for i in range(len(known))])

plaintext = bytes(data[i] ^ key[i % len(key)] for i in range(len(data)))

print(plaintext.decode(errors="ignore"))
```

---

## Результат

После расшифровки в выводе появляется строка с корректным флагом:

```
SplitCTF{XOR_DecryptioN_C0mplete}
```

---

## Флаг

**`SplitCTF{XOR_DecryptioN_C0mplete}`**

---

## Итог

Задание решается стандартной атакой на XOR с известным фрагментом открытого текста.  
Ключ восстанавливается напрямую, после чего расшифровка не представляет сложности.
