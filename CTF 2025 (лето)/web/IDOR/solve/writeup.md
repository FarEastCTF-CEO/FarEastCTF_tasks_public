# Решение

## Идея
В приложении профиль пользователя открывается по параметру id в URL. Сервер не проверяет, что запрашиваемый профиль принадлежит текущей сессии.

## Эксплуатация
1. Войти под учеткой `admin:admin`.
2. Открыть профиль и обратить внимание на id в адресной строке.
3. Перебирать id, запрашивая чужие профили.
4. Для одного из id в пароле лежит Base64, который декодируется в путь `/flag`.
5. Найти пользователя `SuperAdmin` и войти под ним, затем открыть `/flag`.
