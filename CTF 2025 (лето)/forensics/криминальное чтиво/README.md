Участникам необходимо восстановить всю хронологию событий: какие хосты, учетные записи были скомпрометированы в рамках активности злоумышленников. Результатом будет предоставление ответов на следующие вопросы:
1. С какого ip - адреса производилась атака
    Шаблон ответа: FECTF{ip_address}, где ip_address - требуемый ip - адрес.
2. Какое название имел файл “reverse-shell”.
    Шаблон ответа: FECTF{example.extension}, где example - название файла в нижнем регистре, extension - расширение файла.
3. Какой порт слушается на хосте атакующего в момент выполнения файла “reverse-shell”
    Шаблон ответа: FECTF{port_number}, где port_number - номер порта.
4. От имени какого пользователя злоумышленник смог получить доступ
    Шаблон ответа: FECTF{user}, где user - имя пользователя без пробелов в нижнем регистре.
    Пример: FECTF{adm-example}.
    Пример: FECTF{adm}.
    Здесь adm-example и adm - это user.
5. Какая команда была использована для поиска уязвимости на хосте жертвы.
    Шаблон ответа: FECTF{command}, где command - команда оболочки, которая должна быть выполнена. 
    Пример: bash -c "hostname".
    Пример: zsh -c "ifconfig | grep 'inet'".
    Здесь "hostname" и "ifconfig | grep 'inet'" - это "command".
6. Какой файл был загружен с хоста атакующего для повышения привилегий.
    Шаблон ответа: FECTF{CVE-year-number}, где year - представляет год обнаружения или публикации уязвимости, number - является уникальным номером уязвимости.
    Пример: FECTF{CVE-2021-44228}.
7. Какую уязвимость использует злоумышленник для повышения привилегий.
    Шаблон ответа: FECTF{CVE-year-number}, где year - представляет год обнаружения или публикации уязвимости, number - является уникальным номером уязвимости.
    Пример: FECTF{CVE-2021-44228}
8. Каково имя пользователя созданного злоумышленником для закрепления.
    Шаблон ответа: FECTF{user}, где user - имя пользователя без пробелов в нижнем регистре.
9. Какую утилиту использует злоумышленник для внутреннего сканирования с целью определения “живых” хостов.
    Шаблон ответа: FECTF{tool}, где tool - название утилиты без указания версии в нижнем регистре.
    Пример: FECTF{wireshark}
10. Какой ip - адрес использует контроллер домена.
    Шаблон ответа: FECTF{ip_address}, где ip_address - требуемый ip - адрес.
11. Какую утилиту использует злоумышленник для дальнейшего сканирования на уязвимости
    Шаблон ответа: FECTF{tool}, где tool - название утилиты без указания версии в нижнем регистре.
12. Как называется коллекция утилит используемая злоумышленником для эксплуатации найденной уязвимости.
    Шаблон ответа: FECTF{tool}, где tool - название утилиты без указания версии в нижнем регистре.
    Пример: FECTF{wireshark}.
13. Как называется атака, в результате успешного выполнения которой, злоумышленник смог получить доступ до контроллера домена.
    Шаблон ответа: FECTF{protocol_action}, где protocol - название протокола, action - название атаки или метода.
14. Под какой учетной записью злоумышленник смог получить доступ до контроллера домена.
    Шаблон ответа: FECTF{user}, где user - имя пользователя без пробелов в нижнем регистре.
15. Какую атаку использовал злоумышленник после получения доступа до контроллера домена.
    Шаблон ответа: FECTF{attack}, где attack - название атаки или метода.
