# Решение
Под капотом работает утилита qrencode. Текст для преобразования передается в качестве аргумента при вызове: qrencode "$text" ...

При этом становится возможным внедрять какие-либо опции для утилиты, например "-h", в результате чего будет выведена справка по утилите, а не qr-код.

Дальнейшим шагом является поиск опций, позволяющих читать данные из файлов, и такая опция имеется: "-r", остается узнать что именно нам нужно читать.

Из описания нужно уловить отсылку к переменным окружения ("Вы окружены тайной"), доступ к которым можно получить через /proc/self/env.

В итоге, отправив "-r/proc/self/env" на вход, мы получим qr-код, содержащий флаг.

Строка "-r /proc/self/env" не сработает так как будет воспринята одним аргументом, а не двумя отдельными "-r" и "/proc/self/env"
