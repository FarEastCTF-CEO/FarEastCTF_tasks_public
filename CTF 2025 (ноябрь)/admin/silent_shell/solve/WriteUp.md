# Решение

Оболочка совсем ничего не выдает, а также нет доступа в интернет, scp/sftp не работают, но получить какую-то реакцию от системы нужно.

Этого можно добиться с помощью команды exit. 

Путем объединения команд в конвейер с `&& exit` в конце, можно проверять успешность выполнения команды.

Данный способ можно использовать для извлечения флага из файла /app/flag, но это будет медленно и муторно.

Более интересное решение заключается в использовании кода завершения процесса, который представляется 1-м байтом (8 битами - отсылка к бесконечности).

Команда `exit 100` инициирует завершение ssh-сессии и на клиентскую часть будет проброшен код завершения 100, который мы можем спокойно извлечь командой `echo $?` уже в своей ОС.

Таким образом можно посимвольно читать файл и помещать каждый символ в виде числа в качестве аргумента для exit.

Команда `exit $(od -An -t d1 -N1 -j0 /app/flag)` вернет первый символ флага в виде числа.

Команда `exit $(od -An -t d1 -N1 -j1 /app/flag)` вернет второй символ. И так далее.
